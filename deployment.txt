Attender Deployment Guide
=========================

This is a static PWA with no backend. Deployment means serving the built
files from any static hosting provider.


Prerequisites
-------------
- Node.js 18+ installed
- npm installed


Build for Production
--------------------
1. Open a terminal in the project root directory
2. Install dependencies (if not already done):
       npm install
3. Build the production bundle:
       npm run build
4. The output will be in the "dist" folder


Deployment Options
------------------

Option A: Simple Local/Network Hosting
  1. Build the project (see above)
  2. Serve the dist folder with any static file server:
         npx serve dist
  3. Access from any device on the same network using the displayed URL

Option B: GitHub Pages
  1. Build the project
  2. Push the dist folder contents to a gh-pages branch
  3. Or use the gh-pages npm package:
         npm install -D gh-pages
         npx gh-pages -d dist
  4. Enable GitHub Pages in the repository settings

Option C: Netlify
  1. Connect your repository to Netlify
  2. Set build command: npm run build
  3. Set publish directory: dist
  4. Deploy

Option D: Vercel
  1. Connect your repository to Vercel
  2. Framework preset will auto-detect Vite
  3. Deploy (no additional configuration needed)

Option E: Any Web Server (Apache, Nginx, IIS)
  1. Build the project
  2. Copy the contents of the dist folder to your web server's root directory
  3. Configure the server to redirect all routes to index.html
     (required for client-side routing)

     Nginx example:
         location / {
             try_files $uri $uri/ /index.html;
         }

     Apache (.htaccess):
         RewriteEngine On
         RewriteCond %{REQUEST_FILENAME} !-f
         RewriteCond %{REQUEST_FILENAME} !-d
         RewriteRule . /index.html [L]


PWA Requirements
----------------
- HTTPS is required for the service worker to activate (localhost is exempt)
- Add real icon-192.png and icon-512.png files in the public folder before
  building if you want the full install experience on mobile devices


Notes
-----
- All data is stored in the user's browser (IndexedDB). There is no server
  database to configure.
- Clearing browser data will erase all attendance records.
- The service worker enables offline use after the first visit.
