Attender Deployment Guide
=========================

This app uses Netlify Functions + Netlify Blobs for server-side data storage.
It must be deployed to Netlify.


Prerequisites
-------------
- Node.js 18+ installed
- npm installed
- A Netlify account (free tier works)
- Git repository pushed to GitHub (already done: https://github.com/rvpals/attender_pwa.git)


Deploy to Netlify
-----------------

1. Log in to https://app.netlify.com
2. Click "Add new site" > "Import an existing project"
3. Connect to GitHub and select the "attender_pwa" repository
4. Netlify will auto-detect settings from netlify.toml:
     - Build command: npm run build
     - Publish directory: dist
5. Click "Deploy site"
6. Wait for the build to complete (usually under 1 minute)
7. Your site will be live at a *.netlify.app URL


Custom Domain (Optional)
-------------------------
1. In Netlify dashboard > Site settings > Domain management
2. Click "Add custom domain"
3. Follow DNS configuration instructions
4. HTTPS is provisioned automatically


Local Development
-----------------
To test with the Netlify Functions API locally:

1. Install Netlify CLI:
       npm install -g netlify-cli

2. Link your local project to the Netlify site:
       netlify link

3. Run the full dev environment:
       npm run dev:netlify

   This starts Vite on port 5173 and proxies API calls through Netlify Dev,
   giving you access to Netlify Blobs locally.


PWA Requirements
----------------
- HTTPS is required for the service worker (Netlify provides this automatically)
- Add real icon-192.png and icon-512.png files in the public folder before
  building if you want the full install experience on mobile devices


Notes
-----
- Data is stored in Netlify Blobs (server-side key-value store). It persists
  across devices and browsers — not tied to a single browser session.
- The Netlify free tier includes 125,000 function invocations/month, which is
  far more than a single classroom would use.
- The service worker caches the app shell for fast loading, but data always
  comes from the server.
